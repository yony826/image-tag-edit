# 图片标签处理工具 - 开发总结

## 开发日期
2026-03-05

## 项目概述
开发了一个使用本地大模型（Qwen3.5:9B）自动识别图片内容并打标签的工具，支持多标签搜索功能。

## 技术方案

| 技术选型 | 说明 |
|---------|------|
| 编程语言 | Python 3.10+ |
| 桌面框架 | PyQt5 |
| 大模型 | LM Studio + Qwen3.5-9b |
| 数据存储 | **图片EXIF元数据**（直接写入图片文件） |

## 项目结构

```
我的图片处理程序/
├── main.py              # 主程序入口

├── image_manager.py     # 图片管理模块

├── search_engine.py     # 搜索模块
├── metadata_storage.py  # 图片EXIF元数据存储模块
├── requirements.txt     # Python依赖
├── run.bat             # 启动脚本
└── 开发文档.md         # 开发文档
```

- [x] 图片导入（支持 JPG、PNG、BMP、GIF、WebP）

## 遇到的问题及解决方案

### 1. 图片无法显示（显示0张）
**原因**：`get_all_folders()` 只扫描子文件夹，没有扫描根文件夹本身
**解决**：在 `load_images()` 和 `process_images()` 中增加根文件夹的检查

### 2. metadata.json 中 tags 为空仍视为已处理
**原因**：判断逻辑只检查 `None`，没有检查空列表 `[]`
**解决**：修改判断条件为 `if not existing_tags`

### 3. LM Studio 模型崩溃（Channel Error）
**原因**：图片太大（约800KB），模型处理超时
**解决**：在 `tag_generator.py` 中添加图片压缩功能，将图片缩放到最大1024像素，JPEG质量85%

### 4. 标签重复问题
**原因**：模型返回重复标签，代码没有去重
**解决**：在 `_parse_tags()` 方法中添加去重逻辑

### 5. 点击图片崩溃
**原因**：`on_image_clicked` 信号传递参数错误
**解决**：修改信号机制，直接传递 `full_path` 而不是 `QListWidgetItem`

### 6. 标签保存到JSON文件（不适合移动）
**原因**：原方案将标签保存到文件夹的 metadata.json 文件
**解决**：改为保存到图片本身的 EXIF 元数据，使用 piexif 库

### 7. EXIF中文编码问题
**原因**：PIL/piexif 保存中文时编码问题
**解决**：使用 piexif 库的 ExifIFD.UserComment 字段，保存为 UTF-8 编码的 bytes

## 依赖安装

```bash
pip install PyQt5 requests Pillow piexif
```

## 运行方式

1. 启动 LM Studio，加载 Qwen3.5-9b 模型
2. 运行程序：
   ```bash
   python main.py
   ```
   或双击 `run.bat`

## 使用说明

1. 点击 **"打开文件夹"** 选择图片目录
2. 点击 **"生成标签"** 让 AI 自动识别图片
3. 标签会**直接写入图片文件的EXIF元数据**
4. 在搜索框输入标签进行搜索（支持 AND/OR 逻辑）
5. 点击图片查看标签详情
6. 可通过 **"文件" → "导出CSV报告"** 导出标签
